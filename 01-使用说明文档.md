# TraceLogic Python — 使用说明文档

> Hamilton Venus `.trc` 日志解析工具 | Python 重写版
> 原项目：[VerisFlow/TraceLogicLocal](https://github.com/VerisFlow/TraceLogicLocal)（C# WPF）
> 重写日期：2026-03-26

---

## 一、项目背景

Hamilton Venus 自动化工作站在运行方法时，会生成 `.trc` 格式的追踪日志文件。这些日志记录了每一步移液操作的详细信息，包括：

- 操作时间戳
- 吸液/放液的耗材位置
- 每个通道的移液体积
- 吸头使用情况

原版 TraceLogicLocal 是一个 C# WPF 桌面应用，本项目将其完整重写为 Python CLI 工具，**无需安装任何第三方依赖**，可直接在 Linux/macOS/Windows 上运行。

## 三、快速使用

### 3.1 解析文件并打印摘要

```bash
python3 main.py your_run.trc
```

输出示例：

```
文件：your_run.trc
TraceEntry 总行数：248
PipettingStep 总数：32
LiquidTransferEvent 总数：16
```

### 3.2 导出液体转移事件到 CSV

```bash
python3 main.py your_run.trc --export transfers.csv
```

CSV 包含以下列：

| 列名 | 说明 |
|------|------|
| Timestamp | 吸液时间 |
| ChannelId | 通道编号（1-8）|
| SourceLabware | 来源耗材 ID |
| SourcePositionId | 来源孔位 |
| TargetLabware | 目标耗材 ID |
| TargetPositionId | 目标孔位 |
| Volume | 移液体积（µL）|
| TipLabwareId | 吸头架 ID |
| TipPositionId | 吸头位置 |

### 3.3 查看所有日志条目

```bash
python3 main.py your_run.trc --show-entries
```

### 3.4 查看聚合移液步骤

```bash
python3 main.py your_run.trc --show-steps
```

## 四、数据模型说明

解析过程分三层，从底层到高层依次为：

```
.trc 文件原始行
    ↓ 正则解析
TraceEntry（每行日志）
    ↓ Start/Complete 配对聚合
PipettingStep（移液步骤）
    ↓ Aspirate→Dispense 状态机配对
LiquidTransferEvent（液体转移事件）
```

### TraceEntry — 单行日志

| 字段 | 类型 | 说明 |
|------|------|------|
| LineNumber | int | 文件行号（从 1 开始）|
| Timestamp | datetime | 时间戳 |
| Source | str | 日志来源（如 `Hamilton`）|
| Command | str | 命令名称（如 `Aspirate`）|
| Status | EntryStatus | Start / Complete / Error / Unknown |
| Details | str | 详情字符串（含通道信息）|
| RawLine | str | 原始未解析行 |

### PipettingStep — 移液步骤

将 Start/Complete 两行配对为一个步骤：

| 字段 | 类型 | 说明 |
|------|------|------|
| ActionType | PipettingActionType | Aspirate / Dispense / PickupTip / EjectTip |
| StartTime | datetime | 步骤开始时间 |
| EndTime | datetime | 步骤结束时间 |
| Duration | timedelta | 耗时 |
| ChannelActions | list[ChannelAction] | 各通道动作详情 |
| StartLineNumber | int | 起始行号 |

### LiquidTransferEvent — 液体转移事件

将 Aspirate→Dispense 配对为一次完整的液体转移，是最终分析结果：

| 字段 | 类型 | 说明 |
|------|------|------|
| Timestamp | datetime | 吸液时间 |
| ChannelId | int | 通道编号 |
| SourceLabware | str | 来源耗材 |
| SourcePositionId | str | 来源孔位 |
| TargetLabware | str | 目标耗材 |
| TargetPositionId | str | 目标孔位 |
| Volume | float | 体积（µL）|
| TipLabwareId | str | 吸头架 |
| TipPositionId | str | 吸头位置 |

## 五、核心解析逻辑

### 5.1 日志行正则

```
^(?P<timestamp>[\d\- :]+)> (?P<source>.+?) : (?P<command>.+?) - (?P<status>\w+); ?(?P<details>.*)$
```

示例匹配：

```
2024-03-15 09:00:01> Hamilton : Aspirate - Complete; channel 1: Plate1, A1, 50.0 uL
```

### 5.2 通道动作正则

带体积（Aspirate/Dispense）：

```
channel (?P<channel>\d+): (?P<labware>[^,]+), (?P<position>[^,]+), (?P<volume>[\d\.]+) uL
```

不带体积（PickupTip/EjectTip）：

```
channel (?P<channel>\d+): (?P<labware>[^,]+), (?P<position>[^,>]+)
```

### 5.3 液体转移状态机

每个通道（1-8）独立维护状态：

```
PickupTip  → 记录吸头信息
Aspirate   → 记录来源耗材、孔位、体积
Dispense   → 生成 LiquidTransferEvent，清空吸液状态
EjectTip   → 清空吸头信息
```

## 六、API 参考

### TraceFileParser

```python
from tracelogic import TraceFileParser

parser = TraceFileParser()

# 从文件解析
result = parser.parse("run.trc")

# 从内存行解析（适合测试）
result = parser.parse_lines(["2024-03-15 09:00:00> Hamilton : Aspirate - Start; "])

# 访问结果
result.AllEntries       # list[TraceEntry]
result.PipettingSteps   # list[PipettingStep]
result.LiquidTransfers  # list[LiquidTransferEvent]
result.Errors           # list[str]
```

### DataExporter

```python
from tracelogic import DataExporter

# 导出全部默认列
DataExporter.export_to_csv(result.LiquidTransfers, None, "output.csv")

# 导出指定列
DataExporter.export_to_csv(
    result.LiquidTransfers,
    ["ChannelId", "SourceLabware", "TargetLabware", "Volume"],
    "output.csv"
)
```

## 八、与原 C# 版本对应关系

| C# 类/方法 | Python 对应 |
|-----------|------------|
| `TraceFileParser.Parse(path)` | `TraceFileParser.parse(path)` |
| `TraceFileParser.ParseLines()` | `TraceFileParser._parse_entries()` |
| `TraceFileParser.AggregatePipettingSteps()` | `TraceFileParser._aggregate_steps()` |
| `TraceFileParser.CreateLiquidTransferEvents()` | `TraceFileParser._build_transfers()` |
| `DataExporter.ExportToCsv()` | `DataExporter.export_to_csv()` |
| WPF MainWindow | `main.py`（CLI，argparse）|
| `EntryStatus` enum | `tracelogic.enums.EntryStatus` |
| `PipettingActionType` enum | `tracelogic.enums.PipettingActionType` |

主要差异：
- GUI → CLI（WPF 替换为 argparse）
- 无需 .NET 运行时，Python 3.10+ 即可
- 拖拽打开文件 → 命令行参数传入路径
